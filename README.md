## 목적
- 영상 편집시 용량이 크면 Timeline 랜더링에서 시간적 손해가 큼
- **영상을 목적에 맞게 쪼갤 필요가 있음**

## 구성
- 영상 불러오기
	- 재생: 기본 플레이어를 통해서 재생
	- 속성보기: 빠른 영상 속성 확인을 위한 버튼
	- 폴더열기: 파일의 위치 열기
- 시작과 종료
	- 방향키(↑/↓:시분초단위 +-; →/←: 시분초단위 이동)
	- 대상 동영상에 따라서 00:00:00.00에서↓에서 최대 동영상 길이로 이동
	- 종료시각은 시작시각보다 빠를 수 없음
- 저장 파일명
	- 무손실 컷팅이 실행 후 재생버튼 활성화
	- 옵션
		- 속성복사: 촬영일시 + 시작offset계산
		- 넘버링: 저장될 파일 명이 겹칠 경우 넘버링(예: output (2).mp4)
	- 저장위치
		- 동일위치, 하위output폴더, 사용자 지정
- history관리
	- 작업히스토리
		- 히스토리 네이밍: 영상명_시작초HHmmSSCS_작업일시YYYYMMDDHHmmSS
	- 최근 작업 폴더 기억

## 파일 구성
```
mp4-trim
├ dist                  : 빌드 후 stand-alone 프로그램이 생성되는 장소
│  ├ app.ico            : 아이콘 (작업표시줄에 표기하려면 exe와 같은 폴더)
|  ├ mp4-trim.exe       : 실행 파일
│  ├ task_history.json  : [앱 생성] 앱 수행 중 방문한 작업 디렉토리 모음
│  └ trim_history.json  : [앱 생성] 컷팅 작업에 대한 기록
├ build.bat             : Stand-alone 프로그램 빌드를 위한 명령어 모음
├ ffmpeg.exe            : [다운로드] 동영상 헨들링 핵심 프로그램
└ mp4-trim.py           : main code
```

## ffmpeg.exe 공수 (중요!)
1. 다운로드: [Gyan.dev 공식 홈페이지](https://www.gyan.dev/ffmpeg/builds/)
2. 설치
    1. ZIP 다운로드: `ffmpeg-release-essentials.zip` 
    2. 압축 해제 후 `bin`폴더에서 `ffmpeg.exe`를 프로젝트 폴더에 복사
